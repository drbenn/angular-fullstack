Use Cases

In general, local storage is a good choice for storing data that is small, persistent, and does not need to be synchronized with a server.
  • User preferences - styles, fonts, themes
  • Login information - username, password
  • Caching data - images or data previously viewed/used, search results, search keywords!
  • Offline access
  • Game data - player progress, high scores

Login Local Storage Cycle

  1. When the user logs in successfully, store the user's authentication token in local storage.
  2. On each page load, check if there is an authentication token in local storage. If there is, the user is logged in. If there is no authentication token, the user is not logged in and should be redirected to the login page.
  3. When the user logs out, remove the authentication token from local storage.
